Homepage

The homepage is the web page that serves to describe the components that make-up the web application. It includes sections which describe the other components/web pages.

Look and Feel

The homepage sets the main look and feel of the web application which is kept consistent throughout.

The main theme of the web application includes the navigation bar at the top of the page with consistent styling, including icons, layout and colours.

Additionaly, the web pages have a static background with scrolling content.

Hovering the cursor over section will make the border display which gives an interactive feel for the user.

Structure

The homepage's structure is of different sections dedicated to each component of the web application.

Each section is used to describe the other web pages and also contains a link to that web page.

All sections and webpages are fully responsive and resizable to accommodate different screen sizes.

Weather Page

This web page uses chart.js to graph weather forecast information across 5 days in Stoke-on-Trent for Temperature, Humidity and Wind Speed.

Link to Weather

Design

This web page follows the same look and feel as the rest of the web application and is fully responsive/resizbale.

The web page displays three individual charts whith their own buttons and Information. These charts each show a different type of weather data, temperature, humidity and wind speed, over the course of five days.

Key Features

The color pickers show a preview of the current colours that the charts are. As the color changes, the charts colours change in real time using Java Script.

Additionaly, each chart can change to a Bar char, Line chart or pie chart at the click of a button.

Java Script is used for the fuctionality of the charts and data is read and stored using JSON.

Log-in Page

This web page uses php to allow a user to log-in and out of the web application, using PHP and CSV file.

Link to Log-in

Design

This web page follows the same look and feel as the rest of the web application and is fully responsive/resizbale.

The web page uses 3 additional PHP pages to set session values, check login details and change the content that is displayed, depending on who is logged in. For example, the users name is displayed and they are told their admin status.

Key Features

User details are kept in a file called users.csv. When a user tries to log-in, their details are compaired to the details in the CSV file.

After, a user successfully logs in they then can access the secret.php page where they are told thier admin status, name and have the option to log out.

Additionaly, if the logged in user is an administrator, they can also access the publication sumbission page.

Publication Page

This web page uses PHP to store details for publications and allows an administrator to sumbit new publications.

Link to Publication List

Design

The list.php page lists the titles of all the publications in the database. Each title is a clickable link that when clicked takes the user to the display.php page which shows the publication in more detail.

The dislay.php page reads form the result.txt file and displays the information on the selected publication.

Key Features

The list.php page displays difernet content depending on who is logged in. If no user is logged in, it will include a link to the log-in page.

Only an administrator can access the submit.php page and make a new submission. A submission is validated via the required propperty so all fields must be filled before sumbission.

New submissions are written to the result.txt file.
